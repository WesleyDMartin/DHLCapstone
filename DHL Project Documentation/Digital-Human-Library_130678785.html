Summary

This web application has been created for the admins of the Digital Human Library, a non profit organization that supports students from around the world to get access to education and information that they ordinarily will not get. To enhance this learning experience, a virtual reality project was undertaken to deliver a new experience. This time it is going help students learn about different culture. DHL will have a range of cultures that will be supported. It will have a range of commonly asked questions about a culture and the answers to those questions. It gets interesting when anybody can get to experience gathering knowledge in a different way.

This document is targeted for the admin who will be managing the contents of the main cultural application. It will explain what contents does this web application has and how to navigate through this application.

Login Page

This web application starts with a login page. It will allow only the admin or anyone who might have access to the system to be able to login to the application. Upon entering the right username and password, the application will re-direct the user to the main admin page where the user will be able to customize the content.

Admin Page

The admin page is basically a one stop stop for the user/admin. This is where the user will be able to add a new contents that will get added to the actual application. Here is how the page looks at a first glance.

Figure 2: Main Admin Page

The admin page has three different section. The first section shows the supported culture, the second section shows the list of questions that are being supported by the application and the third section shows the answers and the video content related to the question. The proper description has been provided below.

Section1: Supported Culture

This section allows the user to add the list of cultures that will be supported by the DHL virtual cultural experience. The user need to write the name of the culture and click “submit” to be able to add the culture to the database.

Figure 3: List of Supported Culture

After adding the cultures to the database, it will get displayed on the admin page as a list. Since the list of cultures will grow over time and it will be time consuming for the user/admin to scroll through the list, pagination has been added. This will save the user a lot of time to just click next. Each list displays only five cultures at a time.

Figure 4: Selected Culture and Question Section

Section2: Supported Questions

This section allows the user to enter a list of questions that will be supported by the main VR application. To submit the questions, the user need to follow three steps.

  • Select the culture under which the questions will be listed

  • Write the question in the section provided

  • Write the answer related to the question that just has been added

  • Add the link to youtube (2D) video or 360 video

  • Submit

Figure 5: Adding Questions Related to the Culture

Also, since the list of questions will keep on increasing, it would be time consuming for the user to scroll through the questions. To enhance user experience, another pagination has been added there where a list of five questions will be displayed at a time.

Section3: Answer

After adding the list of questions and the answers, the user need to make sure that the answers are being displayed properly. To ensure that the answers are being displayed properly, the user has to select a culture, which will show the list of questions that has been added under the culture. Then the user need to select a question that he/she is trying to find the answer for. After selecting the question, the answer will show up at the bottom of the page in the “Answer” section.

Figure 6: Answer to Selected Question
