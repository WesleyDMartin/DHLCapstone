Summary

This page represents a summary of the major features currently included in our project, as well as rough instructions for how to use them

The Main VR Application

The Virtual Reality Desktop application is the purpose of this platform. Using this platform, users can ask a guide various questions and receive relevant answers from the web-based API which contains a database with all the cultures and questions.

This application is built in Unity, with the user interacting with it using the Oculus controls.

Steps

  • After following the Getting Set Up guide, launch the application from Unity, and if you followed the Oculus Setup Guide, the oculus should automatically load into the headset.

  • The guide will greet you, and prompt you to select a culture.

    • You can hide the cultures at any time by clicking the hide cultures button

  • To look around, simply move your head from side to side in the headset.

  • You can use the right controller to point towards any buttons on the control panel and use the index trigger to select

Asking a Question

At any point in the application, you can ask a question about the culture you have selected! The application will then query the database to look for a similar question about the current culture.

Steps to ask a question:

  • To begin asking a question, simply hold down the “A” button on your right-hand controller.

  • When you let up the “A” button, the system will begin looking for a response to your question.

  • If it finds a response, the guide will display the video and text response, and read out the response audibly.

The Third-Party Tools Interface Service

This custom build interface is how we call Google’s Speech to Text and Text to Speech conversions. It is a C# application that operates a service, listening on an open socket. Whenever it receives a command from the main VR app, it will parse out the instruction, and then pass the command along to either the TensorFlow service or Google's APIs. It can receive instructions to:

  • Convert text to speech, via either Google or .NETs APIs

  • Convert speech to text, via the same services

  • Update the current culture, in which case the current set culture is passed along to TensorFlow

  • Search for a question match, again passed along to TensorFlow

The system will listen for a response from any of the above services, and then it will return the audio clip, text, or matched question back to the Virtual Reality app.

The Natural Language Interpretation Service

As discussed above, this service will listen for incoming requests from the Third Party Tools Interface Service, and make use of the Universal Sentence Encoder to match against a set of questions supplied by the Ruby on Rails API. This list of questions will directly reflect the supported questions list on the Administrative Panel.

When the service receives a message to set the culture, it will query the API for all questions matching that culture, and then use the universal sentence encoder to calculate the mathematical value of the sentence. Whenever a request for a matching question comes in, it will calculate the value of that question in the same way, and look for the closest numerical match. That question will then be returned to the Third Party Tools Interface Service, where it will be passed along to the VR desktop app.

The Administrative Portal

The administrative portal is how the platform administrators can control what content is available in the desktop app. Once logged in, it is as simple as entering a new culture and pressing enter. The culture will automatically be added, and then the user can go ahead and start adding questions. To add a question, simply enter the question itself, the text response, the youtube URL, and the video type.

As a note, our application currently only supports youtube videos. No other video source will be properly displayed. Also, to remove content, simply press the “X” symbol. There is currently no protective prompt, so tread lightly.

Ruby on Rails API

Users will require no direct contact with the API. As a quick summary, it is a Ruby on Rails API, which is hosted alongside the Administrative portal. It is quite simple, simply a list of questions, with a culture ID to tie the cultures to the supported questions.
